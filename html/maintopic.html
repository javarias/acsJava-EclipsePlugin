Overview

The ACS Java Eclipse Plugin was designed to keep track, on you Eclipse Java project, the different set of libraries used during ACS Java module development (ACSROOT, INTLIST and INTROOT).

This Plugin will read the interesting files (.jar and .zip) from the ACS know directory structures and put them as dependencies of your project. However this plugin does not take care of the import order of the Java Build Path, the the user must set this up as required.

Setting up the libraries

Once this plugin is installed, a new preference item will show up in the preferences window.

Clicking in the Installed ACS versions item will display the following panel:

Adding a library of any type is easy. You need to click in any Add Library button to display a new wizard to help you to select a directory containing the ACS and/or ACS related binaries.

The name of the directory selected will automatically set based on the last path segment. This is user modifiable, though.

Setting up ACS libraries in your Java project

Once the libraries has been setup properly in the Eclipse preferences. Then it is possible to setup the libraries used to build a Java project based upon ACS.

In this example an Java Project called DC will be created from existing sources. Originally this project is based on ACS and it require a ACSROOT and a INTLIST to build correctly.

Adding ACS libraries is straightforward since it is integrated with Eclipse JDT framework. To add a new library dependency to the project do the following: get properties for the project, in the properties window click in the Java Build Path list item, then click in the Libraries tab and finally click in the Add Library... button. The following Dialog will show up:

Additionally to the regular options a new set of three options is presented:

  • ACS Installation Directory
  • ACS INSTLIST Directory
  • ACS INTROOT Directory
For this project it is necessary to add an ACS Installation Directory and an ACS INTLIST Directory. Selecting ACS Installation Directory and then next, will show up the previously setup ACS installation list.

Selecting the ACS installation and then finish will add to project Libraries the selected ACS installation directory.

The same procedure must be followed for any INTLIST and INTROOT that need to be added to the project.

Remember to set the right order for these libraries, first the INTROOT, then the INTLIST, the the ACSROOT and at the end the JVM. The user can add more jars, and other libraries. the order how to setup them is responsibility of the user.

At the end the project should build ok.
